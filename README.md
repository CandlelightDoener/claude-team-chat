# Claude Team Chat

A WhatsApp-style chat interface where your [Claude Code](https://docs.anthropic.com/en/docs/claude-code) agents have moderated team discussions — with topic management, reactions, and actionable outcomes.

## What is this?

If you use Claude Code with multiple agents (each in their own folder with a `CLAUDE.md`), this app lets them **talk to each other** in a group chat. A coach agent moderates the discussion, manages topics, and keeps things on track.

You type a topic. The agents discuss. You get an outcome protocol with action items.

## Quick Start

**Prerequisites:** [Node.js](https://nodejs.org/) and [Claude Code](https://docs.anthropic.com/en/docs/claude-code) installed.

### 1. Your folder structure

You need agent folders with `CLAUDE.md` files. Something like:

```
my-project/
├── agent-marketing/
│   └── CLAUDE.md
├── agent-engineering/
│   └── CLAUDE.md
├── agent-design/
│   └── CLAUDE.md
└── CLAUDE.md          ← optional root context
```

### 2. Clone this repo into your project

```bash
cd my-project
git clone https://github.com/CandlelightDoener/claude-team-chat.git chat-app
cd chat-app
```

### 3. Let Claude set it up

```bash
claude
```

That's it. Claude reads the `CLAUDE.md` in the chat-app folder, scans your agent directories, and walks you through creating a `config.json`. It asks about names, roles, teams — you answer, it configures.

### 4. Start chatting

```bash
npm start
```

Open [http://localhost:3333](http://localhost:3333) and type a topic.

## Features

### Moderated Discussions
A coach agent orchestrates the conversation — decides who speaks next, when to dig deeper, when to move on.

### Topic Management
Topics are opened, discussed, and closed. When a topic wraps, it folds into a compact card with:
- Summary of what was discussed
- Outcome / decision
- Action items you can approve or reject — right there in the chat

### Message Reactions
When you chime in during a discussion, relevant agents react with emojis and short comments. Hover to see who said what.

### Outcome Protocol
At the end (or anytime via the summary button), get a structured protocol:
- Topics with context and decisions
- Open points
- Next steps as approachable action items

### Auto Mode
Don't want to wait between turns? Toggle Auto and the discussion runs on its own until the coach needs your input.

## How it Works

```
You → Topic → Coach decides who speaks
                  ↓
              Agent responds (via claude -p in their folder)
                  ↓
              Coach evaluates → next agent / wrap topic / ask you
                  ↓
              Topic wraps → summary + action items
                  ↓
              Next topic or FAZIT → outcome protocol
```

Each agent is called with `claude -p` in their own directory, so they have their full `CLAUDE.md` context. The coach reads all agent profiles and the conversation history to make moderation decisions.

**Zero dependencies.** Pure Node.js HTTP server with Server-Sent Events. Vanilla HTML/CSS/JS frontend.

## Configuration

The `config.json` is generated by Claude during setup. You can also create it manually — see `config.example.json`.

| Field | Required | Description |
|-------|----------|-------------|
| `port` | No | Server port (default: 3333) |
| `root` | No | Path to parent directory relative to chat-app (default: "..") |
| `coach` | Yes | Coach agent config (name, role, dir, color) |
| `agents` | Yes | Map of agent-id → {name, role, dir, color, avatar} |
| `teams` | No | Team groupings (auto-created if omitted) |
| `activeAgents` | No | Default active agents (all if omitted) |
| `contextFile` | No | Path to a context file relative to root |
| `feedbackUrl` | No | POST endpoint for user feedback |

## Avatars

Drop PNG images into `public/images/` and reference them in your config:

```json
"avatar": "/images/my-agent.png"
```

Keep them small (200x200px). Without avatars, the UI shows colored initials — looks fine too.
